# Semana 5: Projeto final

Esta semana foi elaborada para ser o ápice das habilidades e conhecimentos que você adquiriu ao longo do programa, permitindo que você **projete e realize um projeto de pesquisa completo**. Abaixo, apresentamos uma análise detalhada de cada componente:

Planejar e executar um projeto de pesquisa completo:

1. **Desenvolvimento:** Crie uma pesquisa sobre um tema relevante.
    * Isso envolve aplicar seu conhecimento sobre o propósito e as funcionalidades do KoboToolbox, conforme aprendido na [Semana 1](./week-1), onde você aprendeu a **criar e implementar uma pesquisa básica usando o construtor de formulários**.
    * Você deve considerar incorporar elementos de design mais avançados, como a **agrupamento de perguntas** que tenham o mesmo significado ou propósito, e aplicar a lógica de **pular (skip logic)** a esses grupos ou perguntas individuais para controlar sua visibilidade. Consulte a seção "Agrupamento de perguntas e lógica de pular" (\[seção: semana-2#question-grouping-and-skip-logic]).
    * Lembre-se de explorar os diferentes tipos de perguntas disponíveis no KoboToolbox [**diferentes tipos de perguntas disponíveis no KoboToolbox**](./week-3#kobo-question-types).
    * Para eficiência e consistência, você pode utilizar a função "[**Biblioteca de Perguntas**](./week-3#question-library)" para salvar grupos de perguntas como modelos, que podem ser reutilizados em diferentes formulários. Isso é útil para perguntas frequentemente feitas ou listas longas de opções.
    * Como uma boa prática para experimentação e para realizar edições de forma segura, você pode [**criar uma cópia de uma pesquisa existente em um novo projeto completamente diferente**](./week-3#cloning-surveys-for-practice) para manter o projeto original e suas respostas intactas.

2. **Implementação:** Lance o questionário e colete os dados da amostra.
    * Com base nas atividades da primeira semana, você precisará **implementar** o questionário que projetou e testá-lo submetendo dados de amostra.
    * Considere os mecanismos de coleta de dados disponíveis: o aplicativo **Kobo Collect para Android** e as **formulários web (Enketo)**.
    * Você também deve gerenciar o acesso e as permissões dos usuários da pesquisa, o que era um objetivo fundamental na Semana 2 ([semana 2](./week-2)), garantindo uma gestão eficaz da pesquisa.
    * À medida que você avança, pode ser necessário **atualizar seu questionário e reimplantá-lo**, observando como o **controle de versão** funciona para gerenciar as alterações.

3. **Limpeza de Dados:** Limpe os dados para prepará-los para sua análise.
    * Embora o KoboToolbox oferece mecanismos simples, eles frequentemente **não são suficientes** para essa tarefa. Portanto, é provável que você precise recorrer a **ferramentas externas**, como **software de planilhas** (por exemplo, LibreOffice Calc, Microsoft Excel, Google Planilhas) ou ferramentas especializadas de limpeza de dados, como o **OpenRefine**.
    * É crucial estar ciente de que, se você fizer edições no seu formulário após começar a coletar respostas, o KoboToolbox cria **novas colunas para as perguntas atualizadas**, separando efetivamente os dados para diferentes versões. Isso pode tornar a guia de **análise** dentro do KoboToolbox inutilizável, exigindo que você **dependa de ferramentas externas para visualização de dados** e **realinhe manualmente os dados** quando eles parecerem separados.

4. **Análise:** Realize análise de dados utilizando uma ferramenta de planilha e visualize os resultados.
    * realizar **análise de dados básica utilizando planilhas** e aproveitar os recursos de geração de relatórios integrados do KoboToolbox.
    * As principais técnicas envolvem a criação de **tabelas dinâmicas e gráficos para resumir dados**. Embora os materiais de aprendizado fornecidos se concentrem no Microsoft Excel para essas técnicas, os mesmos métodos podem ser aplicados ao Google Planilhas ou ao LibreOffice Calc.
    * Você precisará **exportar os dados coletados em vários formatos**, como CSV e Excel, e explorar como eles aparecem na sua ferramenta de planilha preferida.
    * Lembre-se de que o fluxo de trabalho sugerido assume que a limpeza e visualização de dados ocorrem principalmente fora do KoboToolbox, especialmente se os formulários foram editados durante o processo de coleta de dados, pois a aba de análise interna do KoboToolbox pode se tornar inutilizável.

5. **Documentação:** Elabore um relatório conciso que resuma seu processo e os resultados obtidos.
    * Este último passo serve como um resumo de todo o seu projeto, demonstrando sua compreensão do fluxo de trabalho completo da **KoboToolbox**.
    * Embora as fontes apresentem os passos práticos e as ferramentas para o projeto, implantação, limpeza e análise de dados, elas **não fornecem detalhes específicos sobre o conteúdo ou o formato deste relatório final.**
