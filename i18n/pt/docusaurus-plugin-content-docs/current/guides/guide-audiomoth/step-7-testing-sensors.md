# Apêndice 1: Testes de Sensores Acústicos

Sensores acústicos podem ter qualidade de microfone variável, dependendo de:

*   Danos
*   Condições inconsistentes de fabricação
*   Exposição à umidade/condições ambientais

Recomendamos testar todos os sensores acústicos antes da implantação e retestá-los após qualquer período prolongado de uso, antes de reimplantá-los.

Uma boa maneira de testá-las é utilizando uma gravação de "ruído branco" (por exemplo, [este vídeo do YouTube](https://www.youtube.com/watch?v=1HoNsgxKuqA)).

-   Toque o ruído branco usando um alto-falante com o volume ajustado para "moderado".
    (o volume da voz para conversas em ambientes fechados é bom).

-   Coloque cada gravador no mesmo local (se estiver gravando um por vez) e
    orientation and record a short \~5-10 second clip with the white
    noise playing.

-   Certifique-se de manter o volume do alto-falante constante.

-   Baixe as gravações de cada sensor para uma pasta de teste.

-   Abra a pasta em [Raven]
    Lite](https://www.ravensoundsoftware.com/raven-lite-downloads/) or
    another software that renders spectrograms
    ([Audacity](https://www.audacityteam.org/download/)).

-   Garanta que os dados de todos os sensores sejam consistentes.
    (consistent pattern of noise on the spectrograms for each
    recording).

    ![Raven Lite showing two recordings where the gain or microphone
    quality was low. We would likely hold this sensor back until
    settings that produced similar recordings were
    determined.](/img/guides/guide-audiomoth/am_test_rec_raven.jpg)

::: dica

Os testes podem se tornar parte do fluxo de trabalho de configuração, adicionando testes antes da programação dos sensores e da formatação das cartões de memória. Isso pode ajudar a identificar equipamentos defeituosos precocemente, evitando falhas dispendiosas.

:::
