# Perguntas orientadoras para criar um protocolo de compartilhamento de dados

*Esta informação foi adaptada do artigo "Protocolos de Dados Tecnoculturais" de Foster e Schonwetter, publicado em 2024, produzido para o Programa de Financiamento. [^1]*

## :pergunta: Perguntas para Orientar

### Propriedade e Controle

* Quem detém a propriedade e o controle sobre a plataforma de banco de dados?  
* Onde os dados são armazenados e quem é o proprietário dos servidores?  
* Quais protocolos de privacidade e gestão de dados estão em vigor?

### Consentimento e Acesso

* Como a plataforma garante o consentimento livre, prévio e informado (SPII) para o uso de dados?  
* Os membros da comunidade estão envolvidos de forma significativa nas decisões relacionadas à coleta e armazenamento de dados?

### Uso de Dados e Público-Alvo

* Quem terá acesso aos dados e mapas? (Público em geral, membros da comunidade, ONGs ou governos)  
* Como os dados serão utilizados? (Negociação de terrenos, visibilidade do projeto, planejamento comunitário)

### Proteção de Dados e Segurança

* A plataforma de armazenamento de dados está alinhada com os valores e protocolos de governança da comunidade?  
* Quais são os passos que protegem contra ameaças de segurança cibernética e uso não autorizado de dados?

### Sensibilidade e equidade cultural

* A plataforma leva em consideração questões de equidade de gênero no mapeamento da comunidade?  
* As narrativas e prioridades dos povos indígenas são preservadas na representação de dados?

### Evolução dos Dados e Utilização Futura

* Como os dados podem ser atualizados e quais protocolos permitem que as comunidades alterem as permissões?  
* O que acontece com os dados se a organização responsável for dissolvida?  

## :clipboard: Lista de verificação para consentimento ao compartilhar dados

- [ ] **Autoridade para conceder permissão**  
      Identifique quem na comunidade tem a autoridade para conceder permissões para o compartilhamento de dados.

- [ ] **Escopo do Compartilhamento de Dados**  
      Especifique quais dados podem e não podem ser compartilhados.

- [ ] **Benefícios para a comunidade**  
      Descreva os benefícios para a comunidade ao compartilhar esses dados.

- [ ] **Consciência sobre os riscos**  
      Discuta e documente os riscos potenciais para a comunidade relacionados com o compartilhamento de dados.

- [ ] **Propriedade dos dados**  
      Confirme a propriedade dos dados e qualquer transferência de direitos, se aplicável.

- [ ] **Destinatários dos Dados**  
      Defina quem tem acesso aos dados e para quais finalidades específicas.

- [ ] **Segurança de Dados**  
      Especifique como e onde os dados serão armazenados e gerenciados de forma segura.

- [ ] **Gestão de Dados Pós-Projeto**  
      Descreva o que acontece com os dados após a conclusão do projeto.

- [ ] **Resposta a Incidentes de Vazamento de Dados**  
      Defina os protocolos de resposta em caso de violação de dados ou ameaça à segurança.

- [ ] **Dissolução da Organização**  
      Defina protocolos para a gestão de dados caso a organização terceirizada deixe de existir.

- [ ] **Alterações nas permissões**  
      Defina medidas que permitam à comunidade alterar ou revogar permissões a qualquer momento.
